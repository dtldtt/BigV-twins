# 用户上下文与协作约定（移交文档 3/3）

**最后更新**: 2026-06-10 v0.7

---

## 用户画像

DTL 是一位个人投资者，投资 A 股 / 港股 / 美股。日常工作在阿里巴巴（IdeaLab）。他不是专业金融从业者，但对投资有系统化的学习意愿和实战经验。

### 投资风格
- 偏价值投资 + 趋势跟踪混合
- 关注高股息/红利策略（银行、消费龙头）
- 会做短线博弈（如事件驱动），但正在学习控制冲动
- 持仓 70+ 只股票（分散），正在学习集中
- 每天看 5 位知乎投资博主的更新，周末看投资大师的经典

### 对系统的核心期望
1. **省时间** — 不用逐个看博主帖子，一份 Digest 搞定
2. **学习成长** — 通过 AI 复盘发现自己的行为模式和错误
3. **可操作** — 不要泛泛而谈，要给具体的规则和数字
4. **不编造** — LLM 不知道就说不知道，不要瞎编

## 协作约定（极其重要）

### 沟通流程
1. **不能直接改代码** — 先列出现状，让用户 review，讨论后确认再改
2. **每个 prompt 改动都要测试对比** — 旧 vs 新，给出完整输出和分析
3. **测试报告要完整** — 包含完整的 LLM 输出、量化对比、我的分析和推荐
4. **所有报告存本地** — `~/Downloads/` 下生成 markdown，然后归档到 `docs/`

### Git 约定
- commit 信息用**中文**，简洁说清「为什么」
- **不主动 push**，除非用户明确要求
- **不主动 commit**，除非用户明确要求
- **不用 `--no-verify`** 跳过 hooks
- README 每次大改动后检查是否需要更新
- changelog 在 `/changelog` 页面，每次版本更新后写用户视角的更新公告
- 用 git tag 标记版本（v0.6, v0.7...）

### URL 约定
- 所有生成的链接必须跟随用户的访问方式
- http://8.155.174.112:8001 访问 → 链接用 http + IP
- https://8-155-174-112.nip.io 访问 → 链接用 https + 域名
- zhihu 归档站在同 host 的 :8000 端口

### Prompt 设计原则（用户确认的）
- 身份定义要强（15+ 年经验、具体的专业领域）
- 分红问题必须调 MCP 工具（get_dividend_history），不能让 LLM 编数字
- 博主名用完整知乎用户名，不写死在 prompt 里
- 引用格式：*「原文内容」*（「」包裹 + 斜体）
- 禁止 LLM 自创板块归类（博主说"光模块"不要改成"AI 算力产业链"）
- 分歧分析用"各自成立的前提条件"，不做裁判
- 空节处理：博主没提到的就跳过，不补外部信息

### 敏感文件
- `chats.db` — 含 bcrypt 密码哈希，gitignored
- `.env` — 含 WEB_SECRET_KEY、QODER_PERSONAL_ACCESS_TOKEN，gitignored
- zhihu 项目 — **只读不要改**，每天有 03:01 crawler 在跑

## 用户的思路和决策记录

### 关于 Daily Digest
- 用户核心需求：**不按博主分开看，要一份全局汇总**
- 确定方案：新 prompt × 方案 C（原文+brief 混合输入）× Qoder ultimate
- 方案 C 比纯 brief（方案 A）信息零损耗，比纯原文（方案 B）更快更省
- 博主名不写死在 prompt 里，从输入的 `=== 博主: XXX ===` 分隔符提取
- 速览部分的原文链接由脚本后处理（不让 LLM 生成 URL）

### 关于大师对话
- 用户认为：大师不应该回避 A 股问题，应该主动用框架分析
- 用户认为：大师应该敢于反对用户的投机行为
- 用户认为：每个大师的回答应该一眼能分辨出是谁（差异化）
- Persona 不在 prompt 里写死，由 get_persona 工具返回

### 关于清仓复盘
- 用户核心需求：不只是"对/错"，而是**为什么对/为什么错**，学到什么
- 用户特别关注：越跌越买的行为模式、双顶不卖的纪律问题
- 用户认为：自评对复盘质量是决定性的（测试验证了这一点）
- 确定方案：Enhance 版 prompt + Qoder auto
- 每月 1 号 07:00 自动跑，显示在 `/growth` 页面

### 关于趋势追踪
- 用户初衷：学习大 V 的分析方法，不是做花里胡哨的仪表盘
- 用户想看：大 V 的判断后来被验证了吗？对的话学他的方法，错的话学他怎么止损
- 用户想整合自己的操作和随笔，三条线交织（大 V 观点 × 我的操作 × 市场走势）
- Phase 1 已完成（数据积累），Phase 2-5 等数据积累后再做

### 关于 Token 监控
- Qwen 和 Qoder 必须分开统计
- Qoder 要按模型区分（ultimate / auto / performance）
- 放在 `/admin/cost` 独立页面，不混在 `/report` 里

### 关于 Persona 更新
- 完全自动化，不需要人工审核
- 更新前后自动 git commit（可随时 revert）
- 增量更新不推翻，保留演化记录
