## Bug 1: 大 V 每日总结 markdown 格式 + 原文链接 — 已修复 (a93d4e6)

### 问题
博主日报 `brief_md` 是纯文本格式，重点内容没有加粗/强调，也没有原文帖子的链接。

### 修复内容

**1. Markdown 格式化**（`_render_brief_md` 函数重写）

| 改前 | 改后 |
|------|------|
| `**主要观点**：xxx` 纯文本一行 | `### 主要观点` 分段标题 + 正文 |
| 金句用 `/` 分隔 | 列表格式，每条一行 `- 「...」` |
| 关键事件 `·` 分隔 | 列表格式 `- 事件名` |
| 个股情绪 `\|` 分隔 | 列表 + emoji 标签（📈看多/📉看空/⚠️回避/➖中性）+ confidence |
| 博主操作无加粗 | 操作动词 **加粗**（如 **reduce**） |
| 转向无标记 | 加 **[转向]** 粗体标签 |

**2. 原文归档链接**

- `fetch_blogger_posts_for_day` 加 `archive_url` 字段 → `https://8-155-174-112.nip.io:8000/content/{db_id}`
- `_render_brief_md` 末尾 `### 原文链接` 段，每篇帖子一个链接

**3. 效果验证**

已重新生成 2026-06-04 的 5 个博主日报，新格式清晰分段，金句/事件/情绪/建议各有 `###` 标题，末尾有归档原文链接。可在 `/report` 页面查看。

## Bug 2: 股票详情页 — 分红 + 涨跌 — 已修复 (0f7620e)

### 问题 2a: 分红不按持股数量更新

**根因**: 创建分红条目时 `shares` 由用户手动填写，没有自动根据实际持仓计算。

**案例**: 今世缘 603369
- 实际持股 500 股（100 + 200 + 200），但分红只算了 300 股

**修复**:
- 新增 `_calc_shares_at_date(session, user_id, ticker, record_date)` 函数
- 遍历该 ticker 所有 open/add/sell/close 操作，累加到股权登记日
- 创建分红条目时自动覆盖用户输入的 shares

**验证**: `_calc_shares_at_date(1, '603369', '2026-06-03')` = 500 ✅

### 问题 2b: 缺少股权登记日

**修复**:
- DB: `ALTER TABLE decision_journal ADD COLUMN record_date TEXT`
- ORM: `DecisionJournal` 加 `record_date` 字段
- 创建表单: action=dividend 时显示「股权登记日」日期选择器（JS 动态切换）
- stock.html: 分红条目显示「（登记日 YYYY-MM-DD）」
- 表单标签动态变化: 分红时「决策时间」改为「除权除息日」、「买入价格」改为「每股分红（元）」

### 问题 2c: 当日涨跌不显示

**现状**: `/stock/{ticker}` 头部已有涨跌幅。`/journal` 列表页的持仓卡片只有现价没有涨跌。

**修复**: `journal/list.html` 现价后面加涨跌幅 badge，红涨绿跌。
