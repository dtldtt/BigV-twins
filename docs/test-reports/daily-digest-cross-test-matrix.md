## 逐组合质量分析

### 组合 1：旧 prompt × B（仅原文）— 2,562 字

**优点**：
- 市场全景最详细，包含大宗商品细分（原油/有色/农产品/美债收益率）
- 板块热度排了明确的优先级（由热到冷）
- 分歧分析给了"如何理解"的建议

**问题**：
- 没有今日重点导读，用户必须从头读
- 没有观察清单，看完不知道关注什么
- 博主名缩写不统一（"三人禾""沈同学""派大星"）
- 速览里的链接是编造的 zhihu 个人主页 URL（不是具体文章）
- 分歧分析里 "如何理解" 做了裁判（"如果你是满仓科技股，沈同学的警告更值得重视"）— 旧 prompt 的矛盾点

### 组合 2：旧 prompt × C（原文+brief）— 2,582 字

**优点**：
- 结构与组合 1 相似但更紧凑
- brief 标注帮 LLM 更快定位重点，全景段更有逻辑

**问题**：
- 同样没有今日重点和观察清单
- 博主名缩写（"派大星"而非"派大星皮皮"）
- 同样编造 URL
- 字数超过旧 prompt 的 1500 字限制

### 组合 3：新 prompt × B（仅原文）— 2,311 字 ⭐

**优点**：
- **今日重点 3 句话导读**效果很好 — 情绪/共识/信号一目了然
- **引用格式完全遵循** *「」* 规范
- **博主全名**全部正确（水又三人禾、阳光下的沈同学、MR Dang、派大星皮皮）
- **观察清单 6 条**，每条有具体价位/条件/时间窗口，非常实用
- **分歧用"各自成立的前提"**而非"谁更可信"，消除了矛盾
- 速览无链接（符合设计，由脚本后处理）
- 共识按信号强度排序，最重要的排前面
- 大宗商品子话题保留了（博主有提到）

**问题**：
- 输出最长（2,311 字接近 2,500 上限，但合理）
- output tokens 最多（6,235） — 因为观察清单和引用增加了内容量
- 耗时最长（121s）

### 组合 4：新 prompt × C（原文+brief）— 2,065 字 ⭐⭐

**优点**：
- **今日重点**最精炼 — 一段 3 句，没有用 bullet 列表，读起来更自然
- **引用格式** *「」* 完全遵循
- **博主全名**全部正确
- **观察清单 5 条**，含"电力是否脱钩科技"这个独有的条件判断 — 说明 brief 标注帮 LLM 发现了更深层的观察点
- **分歧分析**用"各自成立的前提"，且分析最清晰
- **最短**（2,065 字）— 信息密度最高
- **最快**（93s）— 和之前 A/B/C 测试一样，方案 C 反而更快
- **output tokens 最少**（3,655） — 最高效

**问题**：
- 大宗商品细节略少于组合 3（因为 brief 里压缩了这部分）
- 重点事件只有 5 条（vs 组合 3 的 6 条），但每条更精准

### 新 prompt 带来的确定性改进

新 prompt 在**所有维度**都优于旧 prompt（不论搭配 B 还是 C）：
- ✅ 今日重点导读（旧 prompt 完全没有）
- ✅ 观察清单（旧 prompt 完全没有）
- ✅ 引用格式 *「」*（旧 prompt 不统一）
- ✅ 博主全名（旧 prompt 经常缩写）
- ✅ 分歧用"前提条件"而非"谁更可信"（消除内部矛盾）
- ✅ 不编造 URL（旧 prompt 编造了 zhihu 个人主页链接）
