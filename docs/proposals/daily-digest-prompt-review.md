## 当前 Prompt 全文

```
【你的身份】
你是一位拥有 15+ 年中国 A 股 / 港股市场实战经验的资深投资研究总监，
同时是跨团队研究协调专家。你擅长从多位分析师/博主的独立研究中，
提炼出**全局画面**、发现**共识与分歧**、识别**关键信号**。

你的读者是一位忙碌的个人投资者，他没时间逐个看每位博主的帖子，
需要你用一篇 digest 让他 **3 分钟掌握今天的全局信息**。

## 输出格式（Markdown，严格按以下结构）

### 一、今日市场全景
综合所有博主观点，用 3-5 段描述当前资本市场的整体画面：
- A 股整体：指数走势、成交量、情绪（恐慌/犹豫/乐观/亢奋）
- 海外市场：美股/港股/黄金/大宗商品的关键动向
- 行业板块热度：哪些板块被多人看好/看空，热度排序
- 资金面：北向资金、融资融券、ETF 申赎等信号（如有提及）
每个判断必须标注来源博主。

### 二、博主共识（2-5 条）
- 共识内容（一句话）
- 谁说了什么（原话引用 + 博主名）
- 对投资决策的含义

### 三、博主分歧（1-3 条）
- 分歧焦点
- 正方 / 反方
- 如何理解这个分歧（哪个更可信）

### 四、今日重点事件与数据
### 五、各博主核心观点速览

## 硬约束
1. 忠于原文 — 不外推、不脑补
2. 标注来源
3. 不编造共识
4. 不强行制造分歧
5. 原文引用用「」包裹
6. 总字数 800-1500 字
```

- 硬约束 3、4 条（不编造共识、不强行分歧）重要

## 二、Qoder Ultimate 评价（48.9s）

### 用户视角

- **「3 分钟」与 800-1500 字 + 5 大节冲突**。1500 字 markdown 认真读要 5-7 分钟。需要 **TL;DR / 一句话定调**（开头 50 字内）
- **缺「行动暗示」层**。投资者真实问题是「今天我该关注什么 / 该不该动」。把博主提到的**触发条件、关键位、待观察事件**单独拎出来
- **缺「与昨日的差异」**。最想知道的是*变化*：哪位博主转向了？共识是新形成还是延续？
- **Section V 和 Section I 大量重复**
- **没有「反共识信号」**。当所有人都看多时，这本身是个信号

### Prompt 工程

- **最大内部矛盾**：Section III "哪个更可信" 与硬约束 "忠于原文、不外推" **直接冲突**
- "**热度排序**" 是模糊指令 — 按提及次数还是语气强度？
- "**资金面（如有提及）**" 这种 optional 字段容易导致幻觉补全
- 没规定**博主名格式**、**空节处理**、**引用长度上限**
- 没禁止 LLM 自创板块归类（博主说"光模块"，LLM 写成"AI 算力产业链"）

### 最重要 3 个改进

1. **解决"忠于原文" vs "做判断"的内在矛盾** — 删掉仲裁措辞，或划出明确的「点评」小节
2. **加 TL;DR + 今日观察清单** — 从"信息聚合"升级到"决策辅助"
3. **去节间重叠 + 收紧字数** — 全景只放事实，共识分歧专责观点，速览只剩链接+标签

## 三、Qoder Performance 评价（75.2s）

### 用户视角

- **缺"今日结论"或"行动信号"入口** — 结构是描述性的，读者看完不知道该做什么
- **Section V 价值存疑** — 本质是 I-IV 材料的重新打包
- **缺少时效性分层** — 博主观点有"今天要看的事""本周逻辑""中期判断"之分，目前不区分

### Prompt 工程

- 身份定义缺关键限定：*分析必须完全基于所提供的博主原文，不引入外部市场信息*
- **最严重矛盾**：Section III "哪个更可信" 与 "不外推" 直接冲突，LLM 行为会不稳定
- 缺**缺省处理指令** — 博主没提海外市场时，LLM 会跳过还是凭知识补写？
- 字数 "800-1500" 双边范围 LLM 很难命中，改为单侧上限更好

### 最重要 3 个改进

1. **加前置"今日三句话"摘要** — 让读者只看 3 行就能决定是否深读
2. **Section III "哪个更可信" 改为 "各自成立的前提条件"** — 消除矛盾且更实用
3. **加缺省处理指令 + 输入边界限定** — 防止博主发帖少的天出现"静默幻觉"

## 五、综合改进方向

根据三方评审，优先级排序的改进清单：

### P0（必须改）
1. **加 TL;DR 前置摘要** — 3 句话：整体情绪 + 最强共识 + 今日最重要事件
2. **解决"忠于原文 vs 做判断"矛盾** — Section III 改为"各自成立的前提条件"
3. **加输入边界限定** — "所有判断必须来自下方博主原文，不得引入外部信息"
4. **加缺省处理** — "博主未涉及的子话题直接跳过"

### P1（强烈建议）
5. **加"今日观察清单"** — 博主提到的关键价位、事件窗口、验证条件
6. **Section V 改为极简** — 每人一行标签 + 原文链接，不复述
7. **字数改为单侧上限** — "不超过 1500 字（不含链接）"
8. **标注博主擅长领域** — 技术面/基本面/宏观配置，帮读者判断权重

### P2（锦上添花）
9. **vs 昨日变化标注** — "某博主今日态度转变""新增共识"
10. **反共识/异常信号** — 全员一致看多/看空时标记
11. **博主名格式统一** + 引用长度上限（≤30字）
12. **重要性排序标准明确** — 有时间窗口 > 多人共提 > 单人强调
