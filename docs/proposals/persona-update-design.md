# 博主 Persona 月度更新方案

**日期**: 2026-06-09

---

## 背景

每位博主的 persona（`personas/*.md`）是他们数字分身的灵魂 — 定义了投资框架、表达习惯、关注领域、典型口头禅。目前这些 persona 是手写的，基于博主的历史内容一次性生成。

问题是：**博主在成长，persona 没有跟着长**。

一个投资博主的风格和关注点是会变化的：
- 三人禾可能从纯技术分析开始关注宏观流动性
- 沈同学可能从全仓进攻转向红利防守
- 派大星可能发展出新的操作体系（如哑铃策略）

如果 persona 不更新，数字分身的回答会越来越"过时" — 用的还是半年前的框架和口头禅，和博主当前的实际风格脱节。

---

## 设计原则

### 1. 增量更新，不推翻

Persona 的核心身份是稳定的。月度更新是**在原有基础上叠加新发现**，而不是重写。就像一个人的性格不会月月变，但他的关注点、措辞、判断框架会随经验演化。

### 2. 区分"风格演变"和"临时反应"

博主在市场暴跌那天说的"全部清仓"不代表他从此变成保守派。月度更新要从一个月的 brief 里提取**持续性的变化**，过滤掉**单日的情绪波动**。

判断标准：
- 连续 2 周以上反复出现的表述 → 可能是风格演变
- 只出现一两次 → 可能是临时反应
- 博主自己说"我现在改变看法了" → 确认是演变

### 3. 记录演化轨迹

不只是覆盖旧 persona，而是在 persona 文件里保留**演化记录**：

```markdown
## 风格演变记录

### 2026-06 更新
- 新增关注：红利/高股息策略（之前以科技成长为主）
- 新口头禅：「管住手就好」（本月出现 8 次）
- 框架补充：开始使用「哑铃策略」（银行+微盘两端配置）
```

这样以后回看，能看到博主的成长轨迹 — 这本身就是投资学习的一部分。

## 输入设计

每月底喂给 LLM 的数据：

### 输入 1：旧 persona（完整）
当前 `personas/{slug}.md` 的全文。~2000 字。

### 输入 2：本月 brief 摘要
从 `blogger_daily_brief` 提取本月所有交易日的：
- `main_view`（主要观点，150-300 字/天）
- `key_quotes`（金句，0-3 条/天）
- `ticker_opinions`（个股情绪 + sentiment + confidence）
- `actions_self_disclosed`（博主自报操作）

预估：22 天 × ~400 字 = ~9000 字。

### 输入 3：本月 ticker_opinion_log 汇总
按板块/个股聚合博主本月的情绪分布：
```
科技板块：bullish 5次 → neutral 3次 → bearish 8次（月内从看多转看空）
银行/红利：neutral 2次 → bullish 10次（持续看好）
```
这个用数据库聚合，不需要 LLM。~500 字。

### 输入 4：本月 digest 中该博主的共识/分歧表现
从 `daily_digest` 里提取该博主本月：
- 参与了哪些共识（和其他博主观点一致的地方）
- 参与了哪些分歧（和其他博主观点矛盾的地方）
~500 字。

**总输入：~12K 字 ≈ ~16K tokens**，Qoder ultimate 完全能处理。

## Prompt 设计

放在 `prompts/persona/update-persona.md`：

```markdown
【你的任务】
你是一位资深投资研究主管，负责维护投资博主的"风格画像"（persona）。
现在是月底，你需要根据该博主过去一个月的实际发言，**更新**他的 persona。

**核心原则**：
- 这是**增量更新**，不是重写。旧 persona 是基础，保留所有仍然准确的内容
- 区分**持续性变化**和**临时反应**：只有连续 2 周以上反复出现的模式才算风格演变
- 保留博主的**原始措辞**：如果他发明了新的口头禅或比喻，用他的原话记录

**输入**：
1. 旧 persona（完整）
2. 本月每日 brief 摘要（main_view + key_quotes + ticker_opinions + 自报操作）
3. 本月情绪分布汇总（按板块）
4. 本月在 digest 中的共识/分歧表现

**输出**：
一份完整的、更新后的 persona 文件（markdown），保持和旧 persona 一样的结构。
在文件末尾的「风格演变记录」段新增一条本月的更新记录。

## 更新时需要检查的维度

### 投资框架
- 核心方法论有没有变化？（如从纯技术面开始加入基本面考量）
- 有没有新的分析工具/指标被频繁使用？
- 风险偏好是否有移动？（如从进攻转向防守）

### 关注领域
- 有没有新的板块/行业进入他的视野？
- 有没有之前关注的板块他不再提了？
- 对 A 股/港股/美股/大宗商品的关注比例有变化吗？

### 表达习惯
- 有没有新的口头禅或标志性表达？（至少出现 3 次以上才算）
- 有没有新的比喻/类比方式？
- 语气是否有变化？（更谨慎？更自信？更自嘲？）

### 操作风格
- 仓位管理策略有没有变化？
- 对止损/止盈的态度有没有演变？
- 交易频率有变化吗？

### 与其他博主的关系
- 他最近是否经常和某位博主观点一致/冲突？
- 这种一致/冲突是否形成了固定模式？

## 输出格式要求

- 保持旧 persona 的章节结构不变
- 在对应的章节里**增补**新内容（用醒目标记，如「⬆ 2026-06 新增」）
- 如果某个旧描述已经不准确了，**修改**它并标注（如「⬆ 2026-06 修正：从 X 变为 Y」）
- 不要删除任何旧内容，即使它已经不完全准确 — 保留并标注"早期风格"
- 末尾的「风格演变记录」段新增本月条目

## 硬约束

1. 不要编造博主没说过的话 — 所有新增的口头禅/观点必须能在输入的 brief 中找到出处
2. 不要把单日的情绪化表达当成风格变化
3. 不要用你自己的语言替换博主的原话 — 保留他的措辞原样
4. 如果本月没有明显变化，输出就是旧 persona + 一条"本月无显著变化"的记录，不要硬凑
```

## 技术实现

### 定时任务

每月最后一天 22:00 执行（确保当月所有交易日的 brief 都已生成）。

### 数据流

```
每月最后一天 22:00
  → 读取旧 persona（personas/{slug}.md）
  → 查询本月 blogger_daily_brief（brief_json）
  → 查询本月 ticker_opinion_log 聚合
  → 查询本月 daily_digest 中的共识/分歧
  → 拼接输入，调 Qoder ultimate
  → 输出新 persona → 写入 personas/{slug}.md
  → git commit（自动提交，便于 diff 查看变化）
```

### 人工审核

LLM 生成的 persona 更新不应该完全自动化 — 应该：
1. 先写入 `personas/{slug}.md.draft`
2. 通知用户（或在 `/admin` 页面展示待审核的 draft）
3. 用户确认后才替换正式文件

这样避免 LLM 错误理解博主风格导致 persona 劣化。

### 文件改动

| 文件 | 改动 |
|------|------|
| `prompts/persona/update-persona.md` | 新建 — 更新 prompt |
| `persona_updater.py` | 新建 — 数据收集 + 调 Qoder + 生成 draft |
| `app.py` | 加 cron job（每月最后一天 22:00） |
| `admin.py` + 模板 | 加 persona draft 审核页面 |

## 示例：更新前后对比

### 更新前（旧 persona 片段）

```markdown
## 关注领域
- 科技股：AI、半导体、光模块，核心持仓方向
- 消费股：白酒、家电，偶尔配置
```

### 更新后

```markdown
## 关注领域
- 科技股：AI、半导体、光模块 — ⬆ 2026-06 修正：本月明确转为短期看空，
  多次提到「主线反弹进入末端」，从核心持仓变为观望
- 消费股：白酒、家电，偶尔配置
- 红利/高股息：⬆ 2026-06 新增 — 本月首次系统性提及，
  「哑铃策略」中银行端成为新的防御配置方向
- 债券 ETF：⬆ 2026-06 新增 — 开始推荐作为「此环境下优于股票」的替代

## 风格演变记录

### 2026-06 更新
- **重大转向**：从科技进攻型转为防守型配置（红利+债券+银行）
- **新口头禅**：「管住手就好」（本月出现 8 次）、「哑铃策略」（5 次）
- **框架补充**：新增「趋势反弹末端」的量价判断体系（天量=顶部区域）
- **风险偏好**：明显从进攻转向防守，多次强调「严控仓位」
- **与其他博主关系**：本月与沈同学形成"科技看空"共识，与三人禾在"是否恐慌"上有分歧
```

## 成本估算

- 每月 5 个博主 × ~16K tokens 输入 + ~3K tokens 输出 = ~95K tokens
- Qoder ultimate 月付，无额外成本
- 人工审核：每月 5 份 draft，每份 5 分钟 ≈ 25 分钟
